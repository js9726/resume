Explain It Like I Am 12

The problem

An AI can sound confident even when it is wrong. In trading, a wrong "GO" or a missed risk flag can teach the wrong lesson.

The simple idea

We give the AI a quiz. Each question is a trading setup. The answer must say GO, WATCH, or PASS, plus the right score and risk words.

What Neon does

Neon is the online database. It already stores real dashboard picks, so the quiz can use real examples instead of fake ones.

What the harness does

The harness checks the answer. If the score is too low, the build should fail, like a school test that needs a passing mark.

Easy mental model: this is a spell-checker for AI trading judgment. It does not trade. It checks whether the AI explanation follows the rules you already decided.

Professional Explanation

The trading brief eval harness is a deterministic quality gate for LLM-assisted trading analysis. It converts real `AListCandidate` rows from Neon Postgres into eval cases and candidate result files, then scores verdict accuracy, Conviction Score range, setup classification, risk flags, forbidden failure modes, and rationale coverage.

The point is not to predict market outcomes. The point is to prevent regressions in the reasoning contract: a pullback should not be failed only because RVOL is low, a breakout should require volume expansion, and a parabolic chase should not be promoted as a clean GO.

1NeonStores real dashboard A-list candidates.
2ExporterReads rows through Prisma and writes local JSON.
3Golden SetDefines expected verdict, tags, flags, and phrases.
4Eval RunnerScores candidate outputs against the expected contract.
5GateFails if aggregate score drops below the threshold.

Serverless PostgreSQL database that stores the dashboard's real A-list trading rows.

What Is `.mjs`, and Why Use It?

What `.mjs` means

`.mjs` means "JavaScript module." It is still JavaScript, but it tells Node.js to use the modern module system where files can say `import fs from "node:fs"`.

What it is typically used for

Build scripts, command-line tools, data exporters, repo automation, and scripts that need clean imports without changing the whole project configuration.

Why not `.js` here?

A `.js` file can mean old CommonJS or modern modules depending on `package.json`. A `.mjs` file removes the guessing and always runs as a modern module.

Why it fits this project

The exporter and eval runner are Node scripts, not React UI components. `.mjs` is a clean fit for automation that imports Prisma, reads files, writes JSON, and exits.

Validation Target

The working validation target for this project is concrete and machine-checkable:

npm run eval:trading-brief
npm run eval:trading-brief:neon
npm run build

The Neon-backed run exports 24 rows from `AListCandidate`, generates local ignored files, and checks that the aggregate score is at least 80%. The last verified result was 100% over 24 generated cases.

Private data guardrail: generated Neon files are kept under `evals/trading-brief/generated/` and ignored by git because they may include real tickers and trading records.
